## 1. 인터넷이란

- 전 세계의 컴퓨터가 서로 연결되어 데이터를 주고받는 네트워크
- 웹 서비스, 이메일, 게임, 스트리밍 등 다양한 서비스의 기반
- 인터넷 통신은 결국 컴퓨터 간 데이터 전송 과정

## 2. 네트워크 계층 구조

### 2.1 OSI 7계층

- 네트워크 통신 과정을 7개의 계층으로 구분한 개념적 모델
- 실제 인터넷에서는 주로 TCP/IP 모델을 사용

| 계층                     | 역할            |
| ------------------------ | --------------- |
| 응용 계층                | HTTP, FTP, SMTP |
| 전송 계층                | TCP, UDP        |
| 인터넷 계층              | IP              |
| 네트워크 인터페이스 계층 | LAN, Ethernet   |

## 3. IP (Internet Protocol)

### 3.1 IP란

- 인터넷에서 컴퓨터를 식별하기 위한 주소 체계
- IP 주소를 기반으로 데이터를 목적지까지 전달

예시

```text
192.168.0.1
8.8.8.8
```

### 3.2 패킷(Packet)

- 인터넷에서 데이터를 전송하는 단위
- 일반적으로 다음 정보를 포함

```text
출발지 IP
목적지 IP
전송 데이터
```

### 3.3 IP의 한계

#### 비연결성

- 상대방의 상태를 확인하지 않고 데이터 전송
- 서버가 종료되어 있어도 패킷 전송 가능

#### 비신뢰성

- 패킷이 중간에 손실될 수 있음
- 패킷 순서가 변경될 수 있음

#### 프로그램 구분 불가

- 동일한 컴퓨터에서 여러 프로그램이 통신 가능
- IP 주소만으로 어떤 프로그램과 통신해야 하는지 구분 불가

## 4. TCP (Transmission Control Protocol)

### 4.1 TCP란

- 전송 계층(Transport Layer) 프로토콜
- IP의 한계를 보완하여 신뢰성 있는 통신 제공

### 4.2 TCP 특징

#### 연결 지향

- 통신 전 3-way Handshake 수행
- 양측의 통신 가능 여부 확인

#### 데이터 전달 보장

- 데이터 수신 여부 확인
- 손실 발생 시 재전송

#### 순서 보장

- 패킷 순서를 보장
- 순서가 변경되면 재조립 후 전달

### 4.3 3-Way Handshake

```text
Client → SYN
Server → SYN + ACK
Client → ACK
```

- TCP 연결 수립 과정

## 5. Port

### 5.1 Port란

- 하나의 컴퓨터 내부에서 실행 중인 프로그램을 식별하기 위한 번호
- IP 주소와 함께 사용

### 5.2 역할

```text
IP Address → 컴퓨터 식별
Port → 프로그램 식별
TCP → 신뢰성 보장
```

예시

```text
192.168.0.10:80
```

```text
192.168.0.10 → 특정 컴퓨터
80 → 특정 애플리케이션
```

### 5.3 대표 포트

| Port   | 프로토콜 |
| ------ | -------- |
| 20, 21 | FTP      |
| 22     | SSH      |
| 25     | SMTP     |
| 53     | DNS      |
| 80     | HTTP     |
| 443    | HTTPS    |

## 6. DNS (Domain Name System)

### 6.1 DNS란

- 도메인 이름을 IP 주소로 변환하는 시스템
- 인터넷의 전화번호부 역할

예시

```text
google.com
naver.com
github.com
```

### 6.2 동작 과정

```text
1. 사용자가 도메인 입력
2. DNS 서버 조회
3. IP 주소 반환
4. 해당 IP로 서버 접속
```

## 7. 전체 통신 흐름

```text
1. 사용자가 google.com 입력
2. DNS 조회
3. 서버 IP 주소 확인
4. TCP 연결 생성
5. 목적지 Port로 요청 전달
6. 서버 응답 수신
```

## 8. 핵심 정리

```text
DNS  → 도메인을 IP 주소로 변환
IP   → 통신할 컴퓨터 식별
TCP  → 신뢰성 있는 데이터 전달
Port → 컴퓨터 내부 프로그램 식별
```
