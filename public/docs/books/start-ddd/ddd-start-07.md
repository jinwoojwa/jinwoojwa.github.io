## 도메인 서비스

하나의 애그리거트만으로 구현하기 어려운 도메인 로직 담당

### 사용하는 경우

- 여러 애그리거트가 함께 필요한 경우
- 특정 애그리거트에 넣기 애매한 도메인 로직
- 외부 시스템 연동이 필요한 도메인 로직

> **NOTE**
>
> 도메인 서비스는
> **"어디에도 속하지 않는 도메인 로직"** 을 구현하기 위한 객체.

---

## 예시

```text
주문(Order)
      │
      ├──────────────┐
      ▼              ▼
회원(Member)      할인 정책(Policy)
                     │
                     ▼
              할인 금액 계산
```

할인 금액 계산은

- Order만으로도 어렵고
- Member만으로도 어려움

↓

도메인 서비스로 분리

```text
DiscountCalculationService
```

---

## 응용 서비스와 역할 분리

```text
Application Service
      │
      ├─ 트랜잭션
      ├─ 흐름 제어
      └─ 도메인 서비스 호출
                │
                ▼
      Domain Service
                │
                ▼
      도메인 로직 수행
```

### 응용 서비스

- 유스케이스 수행
- 트랜잭션 관리
- 도메인 객체 조회
- 도메인 서비스 호출

### 도메인 서비스

- 도메인 로직 수행
- 트랜잭션 처리하지 않음
- 응용 로직 포함하지 않음

> **NOTE**
>
> 로직이 **비즈니스 규칙**이라면 도메인.
>
> 로직이 **흐름 제어, 트랜잭션**이라면 응용 서비스.

---

## 패키지 위치

도메인 영역에 위치

```text
domain
 ├── member
 ├── order
 ├── product
 └── service
      └── DiscountCalculationService
```
