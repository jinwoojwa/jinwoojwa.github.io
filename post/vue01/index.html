Vue.js 핵심 요약 - 기초 개념부터 앱 생성까지

#Vue
2026-03-24

Vue란 무엇인가?

Vue(뷰)는 UI를 구축하기 위한 자바스크립트 프레임워크이다.

표준 HTML, CSS 및 JavaScript를 기반으로 구축되며, 선언적 렌더링과 반응성이라는 두 가지 핵심 기능을 제공한다.

아래는 기본적인 Vue 구조의 예시이다.

js
import { createApp, ref } from 'vue' createApp({ setup() { return { count:
ref(0) } } }).mount('#app')

<div id="app">
  <button @click="count++">Count is: {{ count }}</button>
</div>
  • 선언적 렌더링: Vue는 표준 HTML을 확장한 템플릿 문법을 사용하여, JavaScript 상태를 바탕으로 HTML이 어떻게 보여야 하는지 선언적으로 작성할 수 있게 한다.
  • 반응성: Vue는 JavaScript 상태 변화를 자동으로 추적하고, 변경 사항이 발생하면 DOM을 효율적으로 업데이트한다.



싱글 파일 컴포넌트 (SFC)

Vue 프로젝트에서는 일반적으로 *.vue 확장자를 가진 싱글 파일 컴포넌트(Single-File Component) 형식을 사용한다. 이는 하나의 파일 안에 로직(JS), 템플릿(HTML), 스타일(CSS)을 모두 캡슐화하는 방식이다. 이 구조는 구성 요소의 관심사를 한곳에 모아 관리하기 용이하게 하며, 규모가 큰 프로젝트에서 유지보수성을 크게 높여준다.

vue
<script setup>
import { ref } from 'vue';
const count = ref(0);
</script>

<template>
  <button @click="count++">Count is: {{ count }}</button>
</template>

<style scoped>
button {
  font-weight: bold;
}
</style>



API 스타일

1️⃣ Options API

Options API는 data, methods, mounted와 같은 객체 옵션을 사용하여 컴포넌트의 로직을 정의한다. 정의된 속성은 컴포넌트 인스턴스를 가리키는 this를 통해 접근할 수 있다. 로직이 옵션별로 분리되어 있어 초보자가 구조를 파악하기에 직관적이다.

vue
<script>
export default {
  // 상태(State) 정의
  data() {
    return {
      count: 0,
    };
  },
  // 로직(Methods) 정의
  methods: {
    increment() {
      this.count++;
    },
  },
  // 계산된 속성(Computed) 정의
  computed: {
    doubleCount() {
      return this.count * 2;
    },
  },
};
</script>

<template>
  <button @click="increment">Count: {{ count }}</button>
  <p>Double: {{ doubleCount }}</p>
</template>

2️⃣ Composition API

Composition API는 import한 함수들을 사용하여 컴포넌트의 로직을 정의하는 방식이다. 일반적으로 <script setup>과 함께 사용된다. 로직을 기능별로 그룹화하고 재사용하기에 유리하며, 규모가 큰 애플리케이션에서 복잡성을 관리하는 데 최적화되어 있다.

vue
<script setup>
import { ref, computed } from 'vue';

// 상태 정의 (반응성 주입)
const count = ref(0);

// 계산된 속성 정의
const doubleCount = computed(() => count.value * 2);

// 로직 정의
const increment = () => {
  count.value++;
};
</script>

<template>
  <button @click="increment">Count: {{ count }}</button>
  <p>Double: {{ doubleCount }}</p>
</template>



Vue 애플리케이션 생성

애플리케이션 인스턴스

모든 Vue 애플리케이션은 createApp 함수를 사용하여 새로운 애플리케이션 인스턴스를 생성하는 것에서 시작한다.

이 인스턴스는 앱 전체에 적용될 설정이나 에셋을 등록하는 중심축 역할을 한다.

js
import { createApp } from 'vue';
const app = createApp({});

루트 컴포넌트

createApp에 전달되는 객체는 사실상 컴포넌트다.

모든 앱은 다른 컴포넌트들을 자식으로 포함할 수 있는 루트(Root) 컴포넌트가 필요하며, 일반적으로 App.vue 파일이 그 역할을 수행한다.

js
import { createApp } from 'vue'; // SFC에서 루트 컴포넌트 App을 import import
App from './App.vue'; const app = createApp(App);

애플리케이션 마운트

애플리케이션 인스턴스는 .mount() 메서드가 호출되기 전까지는 아무것도 렌더링하지 않는다.

이 메서드는 실제 HTML 요소(컨테이너)의 셀렉터나 DOM 객체를 인자로 받는다.

주의할 점은 앱의 모든 설정(에러 핸들러, 컴포넌트 등록 등)은 반드시 마운트하기 전에 완료되어야 한다는 것이다.


앱 설정

인스턴스의 .config 객체를 활용하면 애플리케이션 수준의 에러를 처리하거나 전역 속성을 정의할 수 있다.

또한, .component() 메서드를 사용하여 특정 컴포넌트를 전역으로 등록하면 앱 내 어느 곳에서든 별도의 import 없이 해당 컴포넌트를 사용할 수 있다.

js
app.config.errorHandler = (err) => { // 오류 처리 }; app.component('HelloBtn',
HelloBtn);



정리

  • Vue는 싱글 파일 컴포넌트(SFC) 구조를 사용하여 HTML, CSS, JS를 하나의 파일에서 체계적으로 관리하고 캡슐화한다.
  • 모든 앱은 하나의 루트 컴포넌트에서 시작하여 트리 구조로 확장되며, createApp을 통해 생성된 인스턴스가 그 중심 역할을 한다.
  • 생성된 인스턴스는 반드시 특정 HTML 요소에 .mount() 되어야만 실제 화면에 렌더링되며, 마운트 전 모든 전역 설정이 완료되어야 한다.
