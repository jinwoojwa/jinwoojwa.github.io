Vue.js 핵심 요약 - 반응성의 기초 (ref, reactive)

#Vue
2026-04-13

Vue.js의 반응성(Reactivity)이란?

Vue의 가장 큰 특징 중 하나는 데이터가 변경될 때마다 화면이 자동으로 업데이트되는 반응성(Reactivity) 이다.

일반적인 JavaScript 객체는 값이 변해도 DOM을 자동으로 갱신하지 않지만, Vue의 반응성 시스템은 데이터의 변화를 감지하여 연결된 템플릿을 다시 렌더링한다.

Composition API에서는 이를 위해 ref와 reactive라는 두 가지 핵심 API를 제공한다.



ref: 모든 타입에 대응하는 반응성 주입

ref는 인자로 전달받은 값을 내부의 .value 속성에 담고 있는 객체로 반환한다. 기본 자료형(String, Number, Boolean)뿐만 아니라 객체나 배열도 다룰 수 있다.

특징 및 사용법

  • 값 접근 : 스크립트 내에서는 항상 .value를 통해 접근하고 수정해야 한다.
  • 템플릿 활용 : template 영역에서는 .value를 생략하고 변수명 그대로 사용할 수 있다. (자동 언래핑)
JavaScript
import { ref } from 'vue'

const count = ref(0)
const message = ref('Hello World!')

function increment() {
  count.value++ // 스크립트에서는 .value 필수
}



reactive: 객체 전용 반응성 주입

reactive는 객체나 배열과 같은 참조 타입(Reference Type) 전용 반응성 API이다. ref와 달리 내부 값을 .value로 감싸지 않고, 객체 자체를 프록시(Proxy)로 만든다.

특징 및 사용법

  • 값 접근 : 일반 JavaScript 객체를 다루듯 .value 없이 속성에 바로 접근한다.
  • 제한 사항 : 기본 자료형(Number, String 등)에는 사용할 수 없으며, 객체 전체를 새 객체로 교체하면 반응성을 잃어버린다.

reactive는 내부적으로 Proxy로 객체를 감싸 추적하기 때문에, 새로운 객체로 교체하면 기존 Proxy와의 연결이 끊어진다.

JavaScript
import { reactive } from 'vue'

const state = reactive({
  count: 0,
  user: { name: 'kim' }
})

function increment() {
  state.count++ // .value가 필요 없음
}



ref vs reactive: 무엇을 쓸 것인가?

두 방식 중 무엇을 선택할지는 개발 스타일의 영역이지만, 최근 Vue 생태계에서는 다음과 같은 이유로 ref 사용을 권장하는 추세이다.

그 이유는 다음과 같다:

  • reactive는 구조 분해 시 반응성이 깨지기 쉽다.
  • ref는 일관된 방식(.value)으로 상태를 관리할 수 있다.
  • Composition API에서 함수 단위로 상태를 분리할 때 ref가 더 예측 가능하다.

구분refreactive
지원 대상모든 타입 (String, Number, Object 등)참조 타입 (Object, Array, Map 등)
접근 방식스크립트: .value / 템플릿: 자동 언래핑.value 없이 직접 접근
재할당.value를 통해 새 객체 할당 가능객체 자체 재할당 시 반응성 상실
구조 분해 할당반응성 유지 (객체 내 ref 포함 시)반응성 상실 (일반 값으로 변함)
추천 용도단일 값, 기본 자료형, 대부분의 경우관련된 큰 규모의 상태 객체

⚠️ 주의할 점: 구조 분해 할당(Destructuring)

reactive 객체를 구조 분해 할당하면 해당 속성은 일반 상수로 변하며 반응성을 잃게 된다. 반면 ref는 객체 내부에 있더라도 독립적인 반응성을 유지하기에 유리하다.

JavaScript
import { reactive } from 'vue'

const state = reactive({ count: 0 })

// 구조 분해 할당을 하는 순간 반응성을 잃음
let { count } = state

count++ // 변수 값은 1이 되지만, state.count는 여전히 0이며 UI도 변하지 않음

올바른 예시: ref 또는 toRefs 사용

JavaScript
import { reactive, toRefs, ref } from 'vue'

// 1. 처음부터 ref로 선언
const count = ref(0)

// 2. reactive를 유지하며 분해하고 싶을 때 (toRefs 사용)
const state = reactive({ name: 'kim', age: 20 })
const { name, age } = toRefs(state) // 각각 ref가 되어 반응성 유지



ref, reactive : 비유로 이해하기

Vue의 반응성 시스템을 엑셀(Excel)에 비유하면 그 차이가 명확해진다.


1️⃣ ref : 엑셀의 '특정 셀' 하나

ref는 엑셀 시트 위에서 **이름이 지정된 단 하나의 셀(Cell)**과 같다.

  • 값 접근:
    • A1 셀의 값을 바꾸려면 반드시 해당 셀을 클릭해서 수정해야 한다.
    • 이것이 코드에서 .value를 붙여 데이터에 접근하는 과정과 같다.
  • 독립성:
    • 이 셀을 다른 곳으로 복사하거나 참조하더라도, 엑셀은 여전히 그 셀이 어디를 가리키는지 알고 있다.
    • 따라서 구조 분해를 하거나 다른 객체에 담아도 반응성(연결고리)이 유지된다.

2️⃣ reactive: '표(Table) 이름' 그 자체

reactive는 엑셀에서 특정 범위를 '성적표'라는 이름의 표(Table)로 지정한 것과 같다.

  • 값 접근:
    • 성적표.수학, 성적표.영어처럼 표의 이름(객체)을 통해 내부 셀에 직접 접근하고 수정할 수 있다.
    • 별도의 .value 같은 래퍼를 거치지 않으며, 표(구조)와 데이터가 하나의 단위처럼 동작한다.
  • 구조 분해의 위험성:
    • '수학' 점수만 따로 꺼내서 다른 셀에 값으로 붙여넣기를 하면, 그 값은 더 이상 '성적표'와 연결되지 않는다.
    • 이후 원본 '성적표.수학'을 변경해도, 복사된 값은 변하지 않는 단순한 값이 된다.



정리

  • 반응성이란 데이터의 변화를 감지하여 연결된 DOM을 자동으로 업데이트하는 Vue.js의 핵심 시스템이다.
  • ref는 모든 타입의 데이터를 .value라는 껍데기에 담아 관리하며, 어디서든 참조 주소를 유지하는 독립적인 셀과 같다.
  • reactive는 객체 전용 반응성 API로, .value 없이 속성에 직접 접근할 수 있어 문법적으로 간결하지만 객체 단위로만 동작한다.
  • 재할당 시 ref는 .value를 통해 안전하게 값을 교체할 수 있으나, reactive는 객체 자체를 바꾸면 반응성 연결이 끊어진다.
  • 구조 분해 할당 시 reactive는 반응성을 잃으므로, 이를 방지하려면 toRefs를 사용하거나 처음부터 ref를 사용하는 것이 권장된다.
  • 비유하자면 ref는 주소지가 고정된 개별 셀을 클릭하는 방식이고, reactive는 이름 붙은 표 전체를 하나의 영역으로 관리하는 방식이다.
