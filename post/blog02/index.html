Vue + Vite로 개발 블로그 만들기 (2) : 테마 개편 & UX 개선

이번 글에서는 아래 과정을 진행했다.

  • 블로그 테마(CSS 스타일 설정)
  • 위로 가기(Scroll to Top) 버튼 구현

1. 블로그 테마 설정

기존 블로그는 브라우저 기본 스타일만 적용된 상태라 많이 밋밋했다.

때문에 다양한 스타일을 적용해보고 최종적으로 아래의 스타일로 설정하게 되었다.

  • 어두운 배경 적용: 어두운 배경과(#0d1117) 회백색의(#c9d1d9) 글자색으로 설정
  • 폰트 설정: 구글 폰트의 '나눔 고딕 코딩' 폰트를 적용
  • 코드 하이라이팅: highlight.js를 연동하여 마크다운 내 코드 블록 스타일 설정



2. 위로 가기 버튼 구현

글의 내용이 길어지면 사용자가 페이지 상단으로 돌아가기 위해 한참 스크롤해야 하는 불편함이 있다.

이를 개선하기 위해 우측 하단에 고정된 버튼을 추가했다.


💡 중요 구현 사항

  • 버튼 로직을 ScrollToTop.vue 라는 독립된 컴포넌트로 분리
  • 스크롤 감지와 메모리 누수 방지
    • window.scrollY를 활용해 300px 이상 스크롤 시에만 버튼이 나타나도록 구현
    • 이때 onMounted에서 이벤트를 등록하고, onUnmounted에서 제거하도록 하여 메모리 누수를 방지
  • 부드러운 이동과 Vue Transition
    • window.scrollTo({ top: 0, behavior: 'smooth' })를 사용해 부드러운 상단 이동 구현
    • 버튼이 나타날 때 툭 끊기지 않도록 Vue의 <Transition>을 사용해 페이드 인 효과 적용

🧩 주요 코드

javascript
<script setup>
import { ref, onMounted, onUnmounted } from 'vue';

const isVisible = ref(false);

// 스크롤 위치에 따라 버튼 표시 여부 결정
const checkScroll = () => {
  isVisible.value = window.scrollY > 300;
};

// 컴포넌트 마운트 시 이벤트 등록, 언마운트 시 해제
onMounted(() => {
  window.addEventListener('scroll', checkScroll);
});

onUnmounted(() => {
  window.removeEventListener('scroll', checkScroll);
});
</script>

1️⃣ 반응형 상태 관리(ref)

isVisible은 버튼의 '생존 여부'를 결정하는 스위치이다.

Vue의 ref를 사용하여 이 값이 변할 때마다 화면이 자동으로 업데이트되도록 설계했다.


2️⃣ 메모리 누수 방지 (onMounted & onUnmounted)

위 코드에서 가장 중요한 부분이라고 볼 수 있다.

  • onMounted : 컴포넌트가 화면에 나타날 때 브라우저 전체(window)에 스크롤 감시자를 붙임
  • onUnmounted : 사용자가 다른 페이지로 이동하여 컴포넌트가 사라질 때, 붙여두었던 감시자를 반드시 제거해야 함

만약 제거하지 않으면 페이지를 이동할 때마다 감시자가 중복해서 쌓이게 되고,

결국 브라우저의 메모리를 불필요하게 점유하는 메모리 누수 현상이 발생하여

사이트가 느려질 수 있다.



3. 메타 정보 추가 (태그 & 날짜)

글에 대한 메타 정보(태그, 작성 날짜)를 추가했다.

  • Frontmatter 연동 : 마크다운 상단의 메타 데이터를 파싱하여 각 포스트 배열에 태그 리스트와 날짜가 포함되도록 로직을 개선
  • 레이아웃 구성 : 포스트 카드와 상세 페이지 상단에 태그는 왼쪽, 날짜는 오른쪽으로 배치



4. 태그 필터링 구현

블로그에 글이 쌓일수록 원하는 주제의 글을 빠르게 찾는 기능이 중요해진다.

이를 위해 메인 화면 상단에 태그 필터 기능을 추가했다.


💡 주요 설계 방향

  • 컴포넌트 분리
    • 태그 목록을 추출하고 선택하는 로직을 TagFilter.vue로 분리하여 구현했다.
  • URL 쿼리 기반 필터링(?tag=Vue)
    • URL 쿼리를 사용하여, 특정 태그가 선택된 상태의 링크를 공유하거나 새로고침 시에도 상태가 유지되도록 했다.
  • 부드러운 리스트 전환
    • 필터링 시 목록이 바뀌는 느낌을 줄이기 위해 Vue의 <TransitionGroup>을 적용해 애니메이션을 추가했다.

🧩 주요 코드 (TagFilter.vue)

javaScript
const selectTag = (tag) => {
  // 선택된 태그를 URL 쿼리에 반영 (All일 경우 쿼리 삭제)
  router.push({
    query: { tag: tag === 'All' ? undefined : tag }
  });
};

상세 페이지 내의 태그를 클릭했을 때도 router.push를 통해 메인 홈으로 이동하며

자동으로 필터링이 적용되도록 연동하였다.
