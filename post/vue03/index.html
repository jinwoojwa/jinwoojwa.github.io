Vue.js 핵심 요약 - 데이터 가공과 감시 (computed, watch)

#Vue
2026-04-25

데이터의 변화를 다루는 두 가지 방법

ref와 reactive로 반응형 데이터를 선언하여 값을 저장할 수 있으며, 이를 통해 화면을 값에 따라 업데이트 할 수 있다. 하지만 실제 애플리케이션에서는 단순히 값을 저장하는 것 이상으로, 데이터가 변할 때 특정 계산을 수행하거나 사이드 이펙트(API 호출 등)를 실행해야 하는 경우가 많다.

이때 사용하는 핵심 도구가 바로 computed와 watch이다.



computed: 계산된 속성

computed는 기존 데이터를 기반으로 새로운 값을 생성할 때 사용한다.

핵심은 '캐싱(Caching)' 이다. 의존하는 데이터가 변경되지 않는 한, computed는 이전 계산 결과를 재사용한다.

특징 및 사용법

  • 자동 캐싱: 의존성(내부 데이터)이 변하지 않으면 다시 계산하지 않고 이전 값을 즉시 반환한다.
  • 성능 최적화: 무거운 로직을 반복적으로 수행할 필요가 없을 때 매우 효율적이다.
  • 읽기 전용: 기본적으로는 값을 읽기만 하는 것이 원칙이다. (필요 시 getter/setter 구현 가능)
JavaScript
import { ref, computed } from 'vue'

const count = ref(1)
// count가 변할 때만 다시 계산됨
const doubleCount = computed(() => count.value * 2)

console.log(doubleCount.value) // 2



watch: 감시자

watch는 특정 데이터가 변경될 때마다 실행되는 콜백 함수를 정의한다.

computed가 값을 '생성' 하는 데 집중한다면, watch는 값의 변화에 따른 '동작(사이드 이펙트)' 을 수행하는 데 집중한다.

특징 및 사용법

  • 사이드 이펙트 처리: API 호출, DOM 직접 조작 등 값을 반환하지 않는 동작에 적합하다.
  • 비동기 처리: computed와 달리 비동기 처리가 용이하다.
  • 이전 값 확인: 변경 전의 값과 변경된 값을 동시에 인자로 받아 비교할 수 있다.
JavaScript
import { ref, watch } from 'vue'

const question = ref('')

// question 값이 변할 때마다 실행
watch(question, (newValue, oldValue) => {
  console.log(`질문이 '${oldValue}'에서 '${newValue}'로 변경됨`)
})



computed vs watch: 무엇을 쓸 것인가?

구분computedwatch
목적파생된 상태 계산 (데이터 가공)사이드 이펙트 실행 (데이터 감시)
결과값항상 값을 반환해야 함값을 반환할 필요 없음
실행 시점의존성 변경 시 (Lazy)데이터 변경 즉시
성능캐싱 지원 (효율적)로직 수행마다 실행

💡 선택 방법:

  1. 데이터로부터 다른 값을 만들어야 한다면? → computed
  2. 데이터 변화를 감지해 외부 API를 호출하거나 무언가를 변경해야 한다면? → watch



비유로 이해하기: '주문서' 와 '알림 서비스'

  • computed (주문서의 합계): 식당 영수증의 '합계' 금액과 같다. 주문한 메뉴(데이터)가 바뀌지 않으면 합계 금액은 다시 계산할 필요가 없다. 메뉴가 추가될 때만 자동으로 계산된 값을 보여주는 자동 계산기 이다.
  • watch (결제 알림 서비스): 카드 결제가 승인(데이터 변경)되면 즉시 사용자에게 '결제 완료' 문자(사이드 이펙트)를 보낸다. 문자 발송은 영수증의 합계 금액 자체와는 상관없는, 데이터 변화 후 수행해야 할 별도의 행동 이다.



정리

  • computed는 기존 데이터를 가공하여 새로운 값을 만들며, 의존성이 변하지 않으면 이전 값을 재사용하는 방식으로 동작한다.
  • watch는 특정 데이터의 변화를 지켜보다가 그에 따른 외부 동작(API 호출, DOM 조작 등)을 수행한다.
  • 가급적이면 재사용성과 성능이 뛰어난 computed를 우선적으로 사용하고, 비동기 처리가 필요하거나 데이터 변경 후 특정 동작이 발생해야 할 때 watch를 사용한다.
