## URI란 무엇인가

`URI(Uniform Resource Identifier)`는 인터넷 상의 자원을 식별하기 위한 표준 식별자이다.

웹에서 특정 리소스(웹 페이지, 이미지, 파일 등)를 가리키기 위해 사용되는 주소 체계라고 이해할 수 있다.

여기서 특정 자원이란 어떠한 것이든 가능하며, 예를 들어 아래와 같은 것들이 모두 URI로 식별될 수 있다.

- 웹 페이지
- 이미지
- 동영상
- API
- 파일
- ...

<br><br>

## URI, URL, URN의 관계

URI는 크게 두 가지 방식으로 리소스를 식별할 수 있다.

<img src="https://res.cloudinary.com/dqtljpdu0/image/upload/v1774067758/uri_qtdqkr.png" alt="URI" width="400">

<br>

### URL (Uniform Resource Locator)

`URL`은 리소스의 위치를 기반으로 자원을 식별한다.

```
https://www.google.com/search
```

여기서는 다음 정보들을 알 수 있다.

- 어떤 프로토콜을 사용하는지 `(https)`
- 어떤 서버인지 `(www.google.com)`
- 서버에서 어떤 리소스를 요청하는지 `(/search)`

즉, **URL은 리소스가 어디에 있는지**를 알려주는 주소이다.

우리가 브라우저 주소창에 입력하는 대부분의 주소는 URL이다.

<br>

### URN (Uniform Resource Name)

`URN`은 리소스의 이름을 기반으로 자원을 식별한다.

```
urn:isbn:0451450523
```

이 예시는 특정 책의 ISBN을 의미한다.

URN의 특징은 아래와 같다.

- 위치가 바뀌어도 이름은 변하지 않는다.
- 하지만, 리소스의 위치나 액세스 방법에 대한 정보를 제공하지 않는다.

실제 인터넷에서 URN은 자주 사용되지 않으며, 보통 URI와 URL을 거의 같은 의미로 사용하곤 한다.

<br><br>

## URL의 전체 구조

URL은 다음과 같은 문법 구조를 가진다.

```
scheme://[userinfo@]host[:port][/path][?query][#fragment]
```

아래 URL을 예시로 하나씩 살펴보면

```
https://www.google.com:443/search?q=spring+boot&hl=ko
```

<br>

1️⃣ `scheme`

```
https
```

프로토콜을 지정하는 부분으로 어떤 방식으로 리소스에 접근할지 정의한다.

<br>

2️⃣ `userinfo`

```
userinfo@
```

URL에 사용자 인증 정보를 포함하는 방식이다.

```
ftp://user:password@host
```

보안 문제로 인해 현재는 거의 사용되지 않는다.

<br>

3️⃣ `host`

```
www.google.com
```

서버의 도메인 이름 또는 IP 주소를 의미한다.

<br>

4️⃣ `port`

```
:443
```

서버에 접속하기 위한 포트 번호이다.

보통의 경우 생략되며, 생략될 경우 기본 포트를 사용한다.

- HTTP → 80
- HTTPS → 443

<br>

5️⃣ `path`

```
/search
```

서버 내부에서 어떤 리소스를 요청할지 나타내는 경로이다.

예)

```
/shop/buy-iphone/iphone-air
/p/bus/bus-route
```

<br>

6️⃣ `query`

```
?q=spring+boot&hl=ko
```

서버에 전달하는 추가 파라미터이다.

`key=value` 형태이며, `&`로 여러 파라미터를 구분한다.

보통 다음과 같은 정보를 전달할 때 사용된다.

- 검색어
- 필터
- 페이지 번호
- 정렬 조건
- ...

<br>

7️⃣ `fragment`

```
#section1
```

HTML 문서 내부의 특정 위치로 이동할 때 사용되는 값이다.

예)

```
https://docs.example.com/page#section1
```

fragment는 **서버로 전송되지 않는다.**

브라우저가 HTML을 받은 뒤 클라이언트에서만 처리되는 값이다.

<br><br>

## 브라우저에 URL을 입력한다면?

실제로 브라우저 주소창에 다음 URL을 입력했다고 한다면,

```
https://www.google.com/search?q=spring+boot&hl=ko
```

브라우저와 서버 사이에서는 다음과 같은 과정이 일어나게 된다.

<br>

1. DNS 조회

도메인 이름을 IP 주소로 변환한다.

```
www.google.com → 200.200.200.2
```

2. HTTP 요청 메시지 생성

브라우저는 서버로 보낼 HTTP 요청 메시지를 작성한다.

```
GET /search?q=spring+boot&hl=ko HTTP/1.1
Host: www.google.com
```

3. TCP 연결 생성

브라우저는 소켓(Socket)을 사용해 서버와 TCP 연결을 생성한다.

4. TCP/IP 패킷 생성 및 전송

HTTP 메시지는 그대로 전송되는 것이 아니라 TCP/IP 패킷 안에 담겨 인터넷을 통해 전송된다.

5. 서버 처리

서버는 패킷을 받아 내부의 HTTP 요청 메시지를 해석해 처리하며 응답 메시지를 생성한다.

예)

```
HTTP/1.1 200 OK
Content-Type: text/html;charset=UTF-8
Content-Length: 3423
```

그리고 HTML 데이터를 함께 보낸다.

6. 브라우저 렌더링

브라우저는 서버의 응답을 받은 뒤 다음 작업을 수행한다.

- HTML 파싱
- DOM 생성
- 화면 렌더링

이를 통해 최종적으로 우리가 보는 웹 페이지가 화면에 표시된다.

<br><br>

## 정리

- `URI`는 인터넷에서 자원을 식별하기 위한 개념이다.
- `URL`은 리소스의 위치를 기반으로 자원을 식별한다.
- `URL`은 scheme, host, path, query, fragment 등의 구성 요소로 이루어져 있다.
- 브라우저에 URL을 입력하면 `DNS 조회 → HTTP 요청 생성 → TCP/IP 전송 → 서버 응답 → 브라우저 렌더링` 과정을 거쳐 웹 페이지가 표시된다.
