## 인터넷에서 컴퓨터는 어떻게 소통하는가?

우리가 브라우저에서 `https://www.google.com`을 입력하면 우리의 컴퓨터는 인터넷을 통해 구글 서버와 통신한다.

하지만 컴퓨터는 단순히 데이터를 보내는 것이 아니라 여러 단계를 거쳐 데이터를 전달한다.

이 과정을 표준화하기 위해 등장한 모델이 `OSI 7 Layer`이다.

<br><br>

## OSI 7 Layer란 무엇인가?

`OSI 7 Layer`는 표준 프로토콜을 사용하여 다양한 통신 시스템이 통신할 수 있도록 네트워크 통신 과정을 7개의 계층으로 나누어 설명하는 모델이다.

각 계층은 서로 다른 역할을 담당하며 데이터는 계층을 따라 내려가며 전송되고 수신 측에서는 반대로 올라오며 처리된다.

쉽게 말하면 서로 다른 컴퓨터 시스템들이 서로 통신할 수 있도록 하는 표준을 제공하는 모델이다.

<br><br>

## OSI 7 Layer 구조

<img src="https://res.cloudinary.com/dqtljpdu0/image/upload/v1774067757/osiandtcp_ayz4ss.png" alt="OSI 7 Layer" width="600">

OSI 모델은 다음과 같은 7개의 계층으로 구성되어 있다.

| Layer | 이름                        |     PDU     | 주요 프로토콜 및 장비            |
| :---: | :-------------------------- | :---------: | :------------------------------- |
| **7** | **Application** (응용)      |    Data     | HTTP, HTTPS, FTP, SMTP           |
| **6** | **Presentation** (표현)     |    Data     | ASCII, JPEG, MPEG (암호화, 압축) |
| **5** | **Session** (세션)          |    Data     | SSH, TLS, NetBIOS (세션 유지)    |
| **4** | **Transport** (전송)        | **Segment** | TCP, UDP / L4 스위치             |
| **3** | **Network** (네트워크)      | **Packet**  | IP, ICMP, ARP / 라우터           |
| **2** | **Data Link** (데이터 링크) |  **Frame**  | Ethernet, MAC 주소 / 스위치      |
| **1** | **Physical** (물리)         |   **Bit**   | 케이블, 리피터, 허브             |

각 계층은 특정 작업을 처리하고 인접한 계층과 통신한다.

<br><br>

### Layer 7. Application Layer

이 계층에서는 사용자와 직접 상호작용하는 계층이다.
웹 브라우저, 이메일 프로그램 등의 애플리케이션이 통신을 위해 접하게 된다.

애플리케이션 계층 프로토콜에는 `HTTP`, `HTTPS` ,`FTP`, `SMTP`등이 존재한다.

<br>

### Layer 6. Presentation Layer

`Presentation Layer`에서는 데이터의 `변환`, `암호화`, `압축`을 담당하여 데이터를 적절하게 다룰 수 있도록 한다.

서로 다른 인코딩 방법으로 통신하는 두 장치 사이에서 수신 장치의 애플리케이션 계층이 이해할 수 있도록 데이터를 `변환`하는 일을 하며,

`암호화`가 필요한 경우 데이터에 암호화를 추가하거나, 데이터의 암호화를 디코딩하여 애플리케이션 계층으로 넘겨주기도 한다.

또한, 전송 데이터의 양을 최소화하여 통신의 효율, 속도를 높이기 위해 `압축`도 수행한다.

<br>

### Layer 5. Session Layer

컴퓨터 시스템 간 통신 시작/종료를 담당하는 계층이다.
통신의 시작 ~ 종료까지의 시간을 `세션`이라 하며, 세션을 생성, 유지, 종료하는 역할을 한다.

<br>

### Layer 4. Transport Layer

컴퓨터 시스템 간 데이터 전송의 신뢰성과 흐름 제어를 담당하는 역할을 수행한다.
데이터가 손실되거나 순서가 뒤바뀌지 않도록 관리하며, 데이터를 여러 개의 작은 단위(`Segment`)로 나누어 전송한다.

대표적인 프로토콜로는 `TCP`와 `UDP`가 있다.

- `TCP (Transmission Control Protocol)`
  - 연결 지향 방식
  - 데이터 전송의 신뢰성 보장
  - 데이터 전송 순서 보장

- `UDP (User Datagram Protocol)`
  - 비연결 지향 방식
  - 속도가 빠르지만 신뢰성, 순서 보장 X

<br>

### Layer 3. Network Layer

서로 다른 두 네트워크 간 데이터 전송을 가능하게 하는 역할을 수행한다. 데이터를 목적지까지 전달하기 위한 `라우팅(Routing)`을 담당하며 이는 데이터가 목적지까지 어떤 경로로 이동해야 하는지를 찾는 과정을 말한다.

또한 네트워크 계층에서는 전송 계층에서 전달받은 `Segment`에
`IP Header`를 추가하여 `Packet` 형태로 만든다.

대표적인 프로토콜로는 `IP (Internet Protocol)`가 있다.

<br>

### Layer 2. Data Link Layer

`Data Link Layer`는 같은 네트워크(LAN) 내에서 데이터를 전달하는 역할을 한다. 이 계층에서는 물리적인 주소인 `MAC Address`를 사용하여 장치를 식별하고 데이터를 전달한다.

네트워크 계층에서 전달받은 `Packet`에 `MAC Address` 정보가 포함된 헤더를 추가하여 `Frame` 형태로 만든다. 또한 데이터 전송 과정에서 발생하는 오류 제어도 담당한다.

대표적인 기술로는 아래와 같은 것들이 있다.

- `Ethernet`
- `MAC Address`

<br>

### Layer 1. Physical Layer

데이터를 실제 전기 신호로 변환하여 네트워크를 통해 전달하는 계층이다.

이 계층에서는 케이블, 전기 신호 등의 `물리적인` 장치를 이용하여 데이터를 전송하며, 데이터는 0과 1로 이루어진 `Bit` 형태로 전달된다.

LAN 케이블, 광 케이블 등이 이 계층에 해당한다.

<br><br>

## 🔄 데이터 전송의 실제 과정: 캡슐화와 역캡슐화

<img src="https://res.cloudinary.com/dqtljpdu0/image/upload/v1774067758/osi7layer_ww4z0n.png" alt="데이터 전송 과정" width="600">

사용자가 브라우저에 `https://www.google.com`을 입력했을 때, 데이터는 다음과 같은 여정을 거친다.

<br>

### 📤 1. 송신 측: 데이터 캡슐화 (Encapsulation)

사용자의 요청이 하위 계층으로 내려가며 각 단계에서 필요한 정보를 담은 **헤더(Header)** 가 추가되는 과정이다.

1. **Application (L7~L5):** 브라우저가 **HTTP Request** 메시지를 생성한다. 필요에 따라 데이터를 암호화하거나 압축하며, 서버와의 통신 세션을 수립한다.
2. **Transport (L4):** 데이터를 전송 가능한 크기로 분할하고, 신뢰성 보장을 위한 **TCP 헤더** 를 붙여 **Segment** 를 생성한다.
3. **Network (L3):** 목적지 서버까지의 경로 배정을 위해 **IP 헤더(출발지/목적지 IP)** 를 추가하여 **Packet** 을 만든다.
4. **Data Link (L2):** 인접한 장비 간의 물리적 통신을 위해 **MAC 주소** 정보를 포함한 헤더를 붙여 **Frame** 을 형성한다.
5. **Physical (L1):** 최종적으로 모든 데이터를 전기 신호인 **Bit(0과 1)** 로 변환하여 네트워크 케이블을 통해 송신한다.

<br>

### 📥 2. 수신 측: 데이터 역캡슐화 (Decapsulation)

목적지에 도착한 데이터는 다시 상위 계층으로 올라가며 헤더를 하나씩 제거하고 원본 데이터를 복원한다.

1. **Physical (L1):** 케이블로 들어온 전기 신호를 다시 디지털 데이터(Bit)로 복원한다.
2. **Data Link (L2):** 프레임 헤더의 MAC 주소를 확인하여 본인에게 온 데이터인지 검증한 뒤 헤더를 제거한다.
3. **Network (L3):** IP 주소를 확인하여 최종 목적지가 맞는지 체크하고 패킷 헤더를 제거한다.
4. **Transport (L4):** 분할되어 들어온 세그먼트들을 순서대로 **재조립** 하고 전송 헤더를 제거한다.
5. **Application (L7~L5):** 최종적으로 웹 서버 프로그램이 복원된 **HTTP Request** 를 수신하여 사용자 요청을 처리한다.

<br><br>

## 정리

- **OSI 7 계층**은 네트워크 통신 과정을 7개 단계로 표준화하여 서로 다른 시스템 간의 원활한 통신을 돕는 모델.
- 데이터는 상위 계층에서 하위 계층으로 내려가며 각 단계의 헤더를 붙이는 **캡슐화(Encapsulation)** 과정을 거친다.
- 수신 측에서는 물리 계층부터 올라오며 헤더를 제거하고 데이터를 해석하는 **역캡슐화(Decapsulation)** 를 수행한다.
- **4계층(Transport)** 은 TCP/UDP를 통해 전송 신뢰성을 담당하며, 데이터 단위는 **Segment**이다.
- **3계층(Network)** 은 IP 주소를 바탕으로 최적의 경로를 찾는 라우팅을 담당하며, 데이터 단위는 **Packet**이다.
- **2계층(Data Link)** 은 MAC 주소를 사용하여 인접한 장치 간의 데이터 전달을 관리하며, 데이터 단위는 **Frame**이다.
- 결과적으로 우리가 브라우저에 주소를 입력하는 단순한 행위 이면에는, 이 7개 계층의 복잡하고 정교한 상호작용이 존재한다.
