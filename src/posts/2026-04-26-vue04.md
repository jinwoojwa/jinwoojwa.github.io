## 컴포넌트 통신: 데이터의 흐름

Vue 애플리케이션은 컴포넌트들의 트리 구조로 이루어진다. 이때 컴포넌트 간의 데이터 통신은 **"부모는 아래로 전달하고(Props), 자식은 위로 알린다(Emits)"** 는 단방향 데이터 흐름을 원칙으로 한다.

<br>

### 1. Props: 부모에서 자식으로

부모 컴포넌트가 자식에게 데이터를 전달할 때 사용한다. Composition API에서는 `defineProps` 매크로를 사용한다.

```JavaScript
// ChildComponent.vue
<script setup>
// Props 정의
const props = defineProps({
  msg: String,
  count: { type: Number, default: 0 }
})
</script>

<template>
  <p>{{ msg }} / {{ count }}</p>
</template>
```

<br><br>

### 2. Emits: 자식에서 부모로

자식 컴포넌트가 부모에게 이벤트나 상태 변경을 알릴 때 사용한다. Composition API에서는 `defineEmits` 매크로를 사용한다.

```JavaScript
// ChildComponent.vue
<script setup>
const emit = defineEmits(['increment'])

const emitUpdate = () => {
  emit('increment', 1) // 부모에게 1을 더해달라는 신호를 보냄
}
</script>

<template>
  <button @click="emitUpdate">카운트 증가 요청</button>
</template>
```

<br><br>

### 3. 부모 컴포넌트의 통합 예시

자식 컴포넌트를 호출하여 다음과 같이 사용한다.

```JavaScript
// ParentComponent.vue
<script setup>
import { ref } from 'vue'
import ChildComponent from './ChildComponent.vue'

const count = ref(0)
const handleIncrement = (val) => { count.value += val }
</script>

<template>
  <ChildComponent :count="count" @increment="handleIncrement" />
</template>
```

<br><br>

## 컴포넌트의 생명주기 (Lifecycle Hooks)

컴포넌트는 생성(Mount)부터 업데이트(Update), 그리고 소멸(Unmount)까지 일련의 과정을 거친다.

특정 시점에 코드를 실행하고 싶다면 Lifecycle Hooks를 사용한다.

주요 Hook 요약

- `onMounted`: 컴포넌트가 DOM에 마운트된 직후 실행 (API 호출 등에 최적)

- `onUpdated`: 데이터 변경으로 인한 재렌더링 후 실행

- `onUnmounted`: 컴포넌트가 파괴될 때 실행 (타이머 해제, 이벤트 리스너 제거 등)

```JavaScript
import { onMounted, onUnmounted } from 'vue'

onMounted(() => {
  console.log('컴포넌트가 화면에 나타났다!')
})

onUnmounted(() => {
  console.log('컴포넌트가 화면에서 사라진다!')
})
```

<br><br>

## 📚 정리

- `Props`는 부모로부터 데이터를 받는 통로이고, Emits는 부모에게 이벤트를 발생시키는 신호를 보낸다.

- Composition API에서는 `defineProps`와 `defineEmits`를 통해 이를 간결하게 선언한다.

- `Lifecycle Hooks`를 사용하면 컴포넌트의 각 생명주기마다 필요한 로직(API 호출, 메모리 정리 등)을 안전하게 배치할 수 있다.
