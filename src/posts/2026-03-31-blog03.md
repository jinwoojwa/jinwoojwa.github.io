## 1. Cloudinary 연동을 통한 이미지 서빙 최적화

포스트에 고화질 이미지가 늘어날 수록 초기 로딩 속도는 비례해서 느려지게 된다.

기존에는 블로그 글에 들어가는 이미지들을 프로젝트 디렉토리 내에 저장하여

사용했지만, 블로그 글이 늘어감에 따라 관리와 성능 면에서 좋지 않을 것 같았다.

이를 해결하기 위해 이미지 호스팅 서비스인 **Cloudinary**를 도입하고, 렌더링 파이프라인에 최적화 로직을 주입했다.

- **자동 포맷 및 품질 조절**: 모든 이미지 주소에 `f_auto, q_auto` 옵션을 동적으로 추가하여 브라우저 환경에 맞는 최적의 포맷(WebP 등)으로 전송한다.
- **유틸리티 분리**: `optimizeCloudinaryImages` 함수를 통해 마크다운 렌더링 직후 HTML 내의 모든 `img` 태그 주소를 자동으로 변환한다.

```javascript
// utils/cloudinary.js 핵심 로직
export const optimizeCloudinaryImages = (content) => {
  // 1. 접속 기기에 따른 동적 너비 설정 (모바일: 500px, 데스크탑: 800px)
  const isMobile = typeof window !== 'undefined' && window.innerWidth < 768;
  const widthParam = isMobile ? 'w_500' : 'w_800';

  // 2. 최적화 파라미터 조합 (포맷/품질 자동화 + 너비 조절)
  const optimizationParams = `f_auto,q_auto,${widthParam}/`;

  // 3. 정규표현식으로 원본 URL을 최적화 URL로 일괄 치환
  return content.replace(
    new RegExp(BASE_URL, 'g'),
    BASE_URL + optimizationParams,
  );
};
```

## 2. TOC(Table of Contents) 구현

블로그 글을 읽을 때 더욱 좋은 UX 경험을 제공하기 위해 목차 기능을 추가했다.

<img src="https://res.cloudinary.com/dqtljpdu0/image/upload/v1774930426/blog-toc_wfiafh.png" alt="blog-toc">

- **Composable 패턴 (useToc.js)**
  - Vue 3의 Composition API를 활용하여 목차 추출 로직을 분리.
- **Sticky positioning**
  - 사용자가 글을 읽는 동안 목차가 항상 우측에 고정되어 이정표 역할을 하도록 구현.

```javascript
// composables/useToc.js
export function useToc(contentRef) {
  const toc = ref([]);

  const generateToc = async () => {
    await nextTick();
    const contentEl = contentRef.value;
    if (!contentEl) return;

    // 본문 내 H2, H3 태그를 수집하여 계층 구조 생성
    const headers = contentEl.querySelectorAll('h2, h3');
    toc.value = Array.from(headers).map((header, index) => {
      const id = header.id || `header-${index}`;
      header.id = id; // 스크롤 이동을 위한 ID 부여
      return { id, text: header.innerText, level: header.tagName };
    });
  };

  return { toc, generateToc };
}
```

## 3. SEO를 위한 사이트맵(sitemap) 자동 생성

검색 엔진에 글이 잘 노출되려면 `sitemap.xml`이 필수적이다.

하지만 매번 새 글을 쓸 때마다 수동으로 갱신하는 것은 비효율적이라 판단하여, 빌드 시점에 자동으로 사이트맵을 생성하는 스크립트를 작성했다.

`Node.js`의 `fs` 모듈을 사용하여 포스트 디렉토리의 마크다운 파일명(Slug)을 읽어오고, 이를 XML 형식으로 변환하여 `dist` 폴더에 배포되도록 구현했다.

```javascript
// utils/sitemap.js 핵심 로직
const getPostSlugs = () => {
  const files = fs.readdirSync(postsDirectory);
  return files
    .filter((file) => file.endsWith('.md'))
    .map((file) => `/post/${file.replace(/\.md$/, '')}`);
};

const sitemap = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="[http://www.sitemaps.org/schemas/sitemap/0.9](http://www.sitemaps.org/schemas/sitemap/0.9)">
  ${['/', ...getPostSlugs()]
    .map(
      (route) => `
    <url>
      <loc>${hostname}${route}</loc>
      <lastmod>${new Date().toISOString().split('T')[0]}</lastmod>
    </url>`,
    )
    .join('')}
</urlset>`;
```

이제 `package.json`의 빌드 명령어에 `node src/utils/sitemap.js`를 연결해두면, 배포할 때마다 최신화된 사이트맵이 자동으로 생성된다.

이제 새 글을 쓰고 배포할 때마다 구글 로봇이 업데이트된 사이트맵을 읽어가서 자동으로 검색 결과에 반영해 준다.
