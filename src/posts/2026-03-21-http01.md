## 인터넷 (Internet)이란

인터넷이란 전 세계의 컴퓨터들이 서로 연결되어 정보를 주고받는 거대한 네트워크이다.

조금 더 쉽게 비유해보면 다음과 같다.

- 컴퓨터 / 스마트폰 → 집
- 인터넷 선 / 무선망 → 도로
- 라우터 → 택배 기사
- 서버 → 건물
- 데이터 → 택배

이렇게 생각하면 인터넷은 전 세계를 연결하는 거대한 도로망이라고 볼 수 있다.

이 도로망 위에서 컴퓨터들이 데이터라는 택배를 서로 주고받는 것, 이것이 바로 인터넷 통신이다.

인터넷 위에서는 다음과 같은 다양한 서비스들이 동작한다.

- 웹 서비스 (웹사이트)
- 온라인 게임
- 클라우드 서비스
- 이메일
- 스트리밍 서비스

이 모든 서비스는 결국 컴퓨터 간의 데이터 통신을 기반으로 동작한다.

<br><br>

## 인터넷 위에서 두 개의 컴퓨터는 어떻게 소통하는가?

### OSI 7 계층

OSI(Open Systems Interconnection) 모델은 서로 다른 컴퓨터 시스템들이 표준화된 방식으로 통신할 수 있도록 만든 개념적인 네트워크 모델이다.

네트워크 통신 과정을 7개의 계층으로 나누어 설명한다.

<br>

### IP (Internet Protocol)

인터넷에서 컴퓨터끼리 데이터를 주고받기 위해서는 서로의 위치를 알아야 한다.

택배를 보내기 위해 주소가 필요한 것처럼,
컴퓨터 간 통신에서도 주소 역할을 하는 정보가 필요하다.

이때 사용되는 것이 바로 IP 주소(IP Address)이다.

예를 들어 다음과 같은 주소가 있다.

```
192.168.0.1
8.8.8.8
```

위와 같은 주소를 기반으로 컴퓨터들은 서로 데이터를 전달한다.

지정한 IP 주소로 데이터를 전달할 때 `패킷(Packet)`이라는 단위로 데이터가 전송된다. 패킷에는 출발지 IP 주소, 목적지 IP 주소, 그리고 전송할 데이터가 포함된다.

<br>

### IP의 한계

IP 프로토콜만으로는 안정적인 통신을 보장하기 어렵다.

<br>

1️⃣ **비연결성**

IP는 데이터를 보내기 전에 상대방이 준비되어 있는지 확인하지 않는다.

즉, 패킷을 받을 대상이 없거나 서버가 다운되어 있어도 패킷은 그대로 전송된다.

2️⃣ **비신뢰성**

IP는 다음과 같은 상황을 보장하지 않는다.

- 패킷이 중간에 사라질 수 있음
- 패킷이 순서대로 도착하지 않을 수 있음

즉 데이터가 정상적으로 도착했는지 보장하지 않는다.

3️⃣ **프로그램 구분 불가능**

하나의 컴퓨터에는 여러 프로그램이 동시에 네트워크를 사용한다.

예를 들어 한 컴퓨터에서

- 웹 브라우저
- 게임
- 메신저

가 동시에 인터넷을 사용할 수 있다.

하지만 IP 주소만으로는 어떤 프로그램으로 데이터를 보내야 하는지 구분할 수 없다.

<br>

### TCP (Transmission Control Protocol)

IP 프로토콜은 데이터를 전달하는 역할만 수행하기 때문에
데이터의 신뢰성을 보장하지 않는다.

이러한 문제를 해결하기 위해 전송 계층(Transport Layer)에서
TCP(Transmission Control Protocol)가 사용된다.

TCP의 주요 특징은 다음과 같다.

<br>

1️⃣ **연결 지향**

데이터를 보내기 전 먼저 연결을 설정하는데, 이를 `3-way handshake`라 한다.

<img src="/images/3-way-handshake.png" alt="3-way handshake" width="300">

이를 통해 양쪽 모두 통신할 수 있음을 확인하게 된다.

2️⃣ **데이터 전달 보장**

TCP는 데이터가 정상적으로 전달되었는지 확인한다.

만약 패킷이 중간에 사라지면 재전송한다.

3️⃣ **순서 보장**

또한 패킷이 도착한 순서를 확인하여 데이터를 올바른 순서로 전달한다.

<br>

### PORT

이제 IP에서 보장하지 못했던 **데이터의 신뢰성** 문제를 해결할 수 있게 되었다. 하지만, 아직 한 가지 문제가 남아 있다.

바로 하나의 컴퓨터 안에서 **어떤 프로그램이 데이터를 받아야 하는지** 구분할 수 없다는 점이다.

예를 들어 다음과 같은 상황을 생각해 볼 수 있다.

- 웹 브라우저
- 온라인 게임
- 메신저

이 프로그램들은 모두 인터넷을 통해 데이터를 주고받는다. 하지만 IP 주소만으로는 해당 데이터가 어떤 프로그램으로 전달되어야 하는지 구분할 수 없다.

이를 해결하기 위해 사용하는 것이 바로 `Port`이다.

`Port`는 컴퓨터 내부에서 실행 중인 프로그램을 구분하기 위한 번호라고 볼 수 있다.

즉 네트워크 통신에서 각각의 역할은 다음과 같이 구분된다.

```
IP Address → 어떤 컴퓨터인지 식별
Port → 컴퓨터 내부의 애플리케이션(프로그램)을 식별
TCP → 데이터가 신뢰성 있게 전달되도록 보장
```

예를 들어 다음과 같은 주소가 있다고 가정하자.

```
192.168.0.10:80
```

이 의미는 아래와 같다.

```
192.168.0.10 : 특정 컴퓨터
80 : 그 컴퓨터에서 실행 중인 특정 프로그램
```

일반적으로 웹 서버는 80번 포트(HTTP) 또는 443번 포트(HTTPS)를 사용한다.

이처럼 IP 주소와 Port 번호를 함께 사용하면 인터넷에서 특정 컴퓨터 시스템의 특정 프로그램과 통신할 수 있게 되는 것이다.

<br>

### DNS (Domain Name System)

지금까지는 IP 주소를 기반으로 컴퓨터 간 통신이 이루어지는 과정에 대해 정리해 보았다.

하지만 실제로 웹사이트에 접속할 때는 다음과 같이 IP 주소 대신 `도메인 이름`을 사용한다.

```
google.com
naver.com
github.com
```

사람이 기억하기에는 숫자로 이루어진 IP 주소보다 문자로 이루어진 도메인 이름이 훨씬 쉽기 때문이다.

하지만 컴퓨터는 도메인 이름을 바로 이해할 수 없기 때문에 실제로 통신을 하기 위해서는 도메인 이름을 IP 주소로 변환하는 과정이 필요하다.

이때 사용되는 시스템이 바로 **DNS(Domain Name System)**이다.

DNS는 도메인 이름을 IP 주소로 변환해주는 시스템이라고 볼 수 있으며, 전화번호부의 역할을 한다고 생각하면 이해하기 쉽다.

예를 들어 사용자가 브라우저에 다음과 같은 주소를 입력했다고 가정해보자.

```
google.com
```

이 경우 실제 통신 과정은 다음과 같이 이루어진다.

```
1. 사용자가 google.com 입력
2. DNS 서버에 도메인 조회 요청
3. DNS 서버가 google.com에 해당하는 IP 주소 반환
4. 해당 IP 주소를 이용해 서버와 통신
```

<br><br>

## 정리

지금까지 살펴본 인터넷 통신의 흐름을 정리하면 다음과 같다.

```
DNS → 도메인 이름을 IP 주소로 변환
IP Address → 통신할 컴퓨터를 식별
TCP → 데이터의 신뢰성 있는 전달 보장
Port → 컴퓨터 내부의 프로그램 식별
```

즉 우리가 브라우저에서 웹사이트에 접속할 때는 다음과 같은 과정이 이루어진다.

```
1. 도메인 입력 (google.com)
2. DNS 조회
3. 서버의 IP 주소 확인
4. TCP 연결 생성
5. 해당 Port의 프로그램과 통신
```
